# Stage 2: Region Segmentation

> How fold markers split the flat board into distinct regions using a planar graph algorithm.

Before any 3D transformation can happen, the flat board must be divided into regions along the fold lines. Each region will later receive its own fold recipe determining how it bends.

## Cutting Lines

Each fold marker generates **N+1 cutting lines** within its zone (where N = `num_bend_subdivisions`, default 8). These lines are perpendicular to the fold axis and slice through the board:

![Cutting Lines](figures/02_cutting_lines.png)

- **Red lines**: zone boundaries (first and last cutting line of each fold)
- **Blue lines**: interior subdivision lines (for smooth mesh bending)
- **Black crosses**: fold marker centers

With 8 markers and 8 subdivisions each, this produces **72 cutting lines** that partition the board.

## Planar Subdivision Algorithm

The cutting lines, board outline, and hole boundaries form a **planar graph**. The algorithm extracts all enclosed faces (regions):

1. **Collect edges**: outline segments + hole boundaries + cutting lines (clipped to board interior)
2. **Build vertex adjacency**: at each vertex, edges are sorted by angle
3. **Trace faces**: starting from an edge, follow the "next clockwise edge" rule at each vertex until returning to the start

### Face Tracing: Next Clockwise Edge Rule

At each vertex, the algorithm picks the **next edge in clockwise angular order** relative to the incoming direction:

![Face Tracing](figures/02_face_tracing.png)

The incoming edge is reversed (gray dotted ray), then the algorithm sweeps clockwise to find the next adjacent edge. This guarantees correct face boundaries even at complex junctions.

## Resulting Regions

After face tracing and filtering, the H-shape board splits into **73 regions**:

![All Regions](figures/02_all_regions.png)

Each region is a closed polygon (colored uniquely, labeled with its index). The narrow strips between cutting lines are individual bend-zone sub-regions that will be mapped onto cylindrical arcs.

## Region Classification

Each region is classified relative to each fold marker based on its representative point:

![Region Classification](figures/02_region_classification.png)

| Color | Classification | Meaning |
|-------|---------------|---------|
| **Green** | Flat (BEFORE) | Not affected by any fold - stays in the original plane |
| **Orange** | IN_ZONE | Inside a bend zone - will be mapped onto a cylinder |
| **Blue** | AFTER | Past a bend zone - continues on a rotated plane |

For the H-shape: 1 flat anchor region, 64 IN_ZONE bend sub-regions, 8 AFTER regions.

## Winding Order Filtering

The face tracing produces both valid board regions (CCW winding) and the outer/infinite face (CW winding). The **signed area** determines which to keep:

![Winding Order](figures/02_winding_order.png)

Using the **shoelace formula**:

```
A = (1/2) * sum(x_i * y_{i+1} - x_{i+1} * y_i)
```

- **A > 0** (CCW): valid board material -- keep
- **A < 0** (CW): outer boundary face -- discard

Regions inside holes are also filtered out using point-in-polygon tests.
