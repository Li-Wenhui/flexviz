# Stage 3: Fold Recipes

> How BFS traversal from the anchor region assigns each region a recipe specifying which folds affect it, in what order, and from which side.

A fold recipe is a list of `(fold, classification, entered_from_back)` tuples that tells the 3D transformation exactly how to bend a region. The recipe is computed via breadth-first search on the region adjacency graph.

## Region Adjacency Graph

Regions that share a boundary edge are **adjacent**. This forms a graph where BFS can propagate fold information:

![Adjacency Graph](figures/03_adjacency_graph.png)

- **Nodes**: regions positioned at their centroid, colored by recipe length (number of folds)
- **Edges**: shared boundaries between adjacent regions
- **Red star**: anchor region (recipe = empty, 0 folds)
- Darker colors = more folds in the recipe

The **anchor region** is chosen as the region that is BEFORE all folds (or affects the fewest folds). It serves as the reference -- the flat, untransformed origin of the board.

## Recipe Buildup

As BFS traverses from anchor to neighbors, each fold crossing adds an entry to the recipe:

![Recipe Buildup](figures/03_recipe_buildup.png)

Starting from the anchor (empty recipe `[]`), each step across a fold boundary adds a new entry:

```
Anchor:  []
Cross fold 0 boundary:  [(F0, IN_ZONE)]
Past fold 0:            [(F0, AFTER)]
Cross fold 1 boundary:  [(F0, AFTER), (F1, IN_ZONE)]
Past fold 1:            [(F0, AFTER), (F1, AFTER)]
...
```

When BFS moves from an IN_ZONE sub-region to the next sub-region past the fold, the entry is **upgraded** from IN_ZONE to AFTER.

## Point Classification

Each point in 2D space is classified relative to a fold using the **perpendicular distance** from the fold center:

![Classification Diagram](figures/03_classification_diagram.png)

```
perp_dist = (point - fold.center) . fold.perp
hw = zone_width / 2
```

| Zone | Condition | Meaning |
|------|-----------|---------|
| **BEFORE** | `perp_dist < -hw` | Not yet reached the fold |
| **IN_ZONE** | `-hw <= perp_dist <= hw` | Inside the bend zone |
| **AFTER** | `perp_dist > hw` | Past the fold, on a rotated plane |

The full H-shape board colored by recipe depth (number of folds affecting each region):

| **Green** | 0 | Anchor region (flat, no folds) |

Recipe depth increases as you move away from the anchor through the fold network. The compact labels (e.g., "A0,Z1") show the recipe entries: A = AFTER, Z = IN_ZONE, followed by the fold index.

## Back-Entry Regions

In complex multi-fold boards like the H-shape, BFS may reach a fold from its **AFTER side** (geometrically past the fold). This is called **back-entry**:

![Back-Entry Regions](figures/03_back_entry.png)

- **Red regions**: entered from the AFTER side (`entered_from_back = True`)
- **Gray regions**: normal entry from the BEFORE side

Back-entry occurs when the BFS path goes around a fold through other folds. The `entered_from_back` flag tells the 3D transformation to **mirror** the cylindrical mapping (the fold axis is at the opposite boundary).

### Back-Entry Detection

```
class_from = classify(region_from, fold)  # Where BFS is coming from
class_to = classify(region_to, fold)      # Where BFS is going

Normal entry:  class_from == BEFORE, class_to == IN_ZONE or AFTER
Back entry:    class_from == AFTER,  class_to == IN_ZONE or BEFORE
```
