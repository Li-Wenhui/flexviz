# Stage 1: Board Import

> What is extracted from the `.kicad_pcb` file and how raw KiCad structures become internal representations.

The parser reads the S-expression file format, extracts geometry from specific layers, and builds the `BoardGeometry` dataclass that feeds the rest of the pipeline.

## Board Outline

The board shape is defined by `gr_line` and `gr_arc` segments on the **Edge.Cuts** layer. These are chained into a closed polygon.

![H-Shape Board Outline](figures/01_board_outline.png)

The H-shape board has ~30 vertices with rounded corners (arcs linearized to short segments). Coordinates are in millimeters, with KiCad's Y-down convention.

## Layer Stack

KiCad organizes data into layers. Each layer contributes different elements to the pipeline:

![Layer Stack](figures/01_layer_stack.png)

| Layer | What it provides |
|-------|-----------------|
| **Edge.Cuts** | Board outline polygon |
| **F.Cu** | Front copper traces and pads |
| **B.Cu** | Back copper traces and pads |
| **User.1** | Fold markers (dotted lines + dimension annotations) |

The board thickness (typically 1.6mm for rigid, 0.1-0.2mm for flex) is extracted from the `(general (thickness ...))` S-expression.

## Traces and Pads

Copper traces are line segments with width, on F.Cu or B.Cu. Pads have shape (circle, rect, oval), size, rotation, and optional drill holes for through-hole components.

![Traces and Pads](figures/01_traces_and_pads.png)

- **Red lines**: F.Cu traces (front copper)
- **Blue lines**: B.Cu traces (back copper)
- **Green dots**: SMD pads
- **Large dots with circles**: Through-hole pads (circle = drill hole)

## Fold Marker Anatomy

A fold marker on User.1 consists of three elements:

![Fold Marker Anatomy](figures/01_fold_marker_anatomy.png)

1. **Two parallel dotted/dashed lines** (red line A, blue line B) defining the bend zone boundaries
2. **A dimension annotation** between them specifying the bend angle in degrees
3. The **center** is the midpoint between the two line midpoints

From these, we derive:

| Parameter | Formula | Description |
|-----------|---------|-------------|
| `zone_width` | perpendicular distance between lines A and B | Arc length of the bend |
| `axis` | unit vector along the fold lines | Rotation axis direction |
| `perp` | `(-axis_y, axis_x)` | Perpendicular to axis (bend direction) |
| `angle` | from dimension text, converted to radians | Bend angle (positive = toward viewer) |
| **`radius`** | **`R = zone_width / \|angle\|`** | **Bend radius** |

## All 8 Fold Markers

The H-shape board has 8 fold markers at various positions and angles:

![All Fold Markers](figures/01_all_fold_markers.png)

- **Green arrows**: fold axis direction (along the fold line)
- **Orange arrows**: perpendicular direction (direction of bending)
- **Dashed lines**: bend zone boundaries for each marker
- **Labels**: angle in degrees

### Axis Normalization

To ensure consistency across parallel folds, the axis direction is normalized:
- **Mostly horizontal folds** (|axis_x| >= |axis_y|): force axis to point in **+X**
- **Mostly vertical folds** (|axis_y| > |axis_x|): force axis to point in **+Y**

This guarantees that the perpendicular direction `perp = (-axis_y, axis_x)` is consistent for parallel folds.

## Key Formulas

![Formulas Reference](figures/01_formulas.png)

| Formula | Meaning |
|---------|---------|
| R = w / \|&theta;\| | Bend radius from zone width and angle (radians) |
| perp = (-axis_y, axis_x) | Perpendicular vector (90 CCW rotation of axis) |
| w = \|line_A - line_B\|_perp | Zone width = perpendicular distance between boundary lines |
